# Cloud Security & Privacy Awareness Programme

**Document ID:** CLOUD-AWR-001 | **Version:** 1.0 | **Classification:** Internal

## 1. Objectives

- Build awareness of cloud-specific risks (mis-configuration, identity, sub-processors, shadow IT).
- Train PII-handling personnel on ISO 27018 obligations (purpose limitation, sub-processor controls, breach notification, DSR support).
- Reinforce shared responsibility — what the CSP does versus what the CSC must operate.

## 2. Audiences and Curricula

| Audience | Topics | Frequency |
|----------|--------|-----------|
| All staff | Cloud AUP, phishing in the cloud era, MFA hygiene | Annual + onboarding |
| Engineering / DevOps | Secure cloud design, IaC guardrails, secrets management, BYOK | Annual + role change |
| Service owners | Shared responsibility, monitoring, incident response, exit | Annual |
| PII-handling staff | ISO 27018 obligations, DSR, breach notification, sub-processor controls | Annual + on regulatory change |
| Cloud admins / SRE | Hardening, segregation, admin operational security (CLD.12.1.5) | Annual |
| Procurement / Legal | CSA / DPA clauses, sub-processor flow-down, transfer mechanisms | Annual |

## 3. Delivery
- E-learning modules (LMS), live workshops, runbook walk-throughs, phishing simulations.

## 4. Records and Evidence
- LMS completion reports, attendance logs, knowledge-check scores, sign-offs.

## 5. Effectiveness
- KPI: ≥ 95% completion within 30 days of due date.
- KPI: phishing failure rate < 5%.
- KPI: cloud-related security incidents linked to user error trending down quarter-on-quarter.

---

*Owner: CISO (with HR partnership) | Review cycle: Annual*
