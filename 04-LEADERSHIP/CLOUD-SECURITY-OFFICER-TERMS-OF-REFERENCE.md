# Cloud Security Officer — Terms of Reference

**Document ID:** CLOUD-CSO-TOR-001 | **Version:** 1.0 | **Classification:** Internal | **Owner:** CISO | **Review:** Annual

## 1. Purpose

Defines the role, mandate, authority, accountability and reporting line of the Cloud Security Officer (CSO) — the executive responsible for cloud-specific information security and the Cloud ISMS aligned with ISO/IEC 27017 and ISO/IEC 27018.

## 2. Reporting Line

- Reports to: **Chief Information Security Officer (CISO)**
- Functional dotted-line: Chief Technology Officer (CTO), Data Protection Officer (DPO)
- Escalation path: CISO → CEO → Board Risk Committee

## 3. Scope of Authority

The CSO has authority to:

- Approve or reject onboarding of new cloud services / providers / regions
- Mandate changes to cloud configurations to mitigate identified risks
- Suspend cloud workloads in the event of confirmed compromise
- Approve sub-processor additions (jointly with DPO for PII)
- Sign off cloud SoA and Risk Treatment Plan
- Represent the organisation to CSPs on security topics

## 4. Key Responsibilities

### 4.1 Strategy & Governance
- Maintain the Cloud Security Policy and Cloud PII Policy
- Own the Statement of Applicability and Risk Treatment Plan for cloud
- Chair the Cloud Security Working Group (monthly)

### 4.2 Risk Management
- Operate the Cloud Risk Assessment Methodology
- Maintain the Cloud Risk Register
- Approve risk acceptances within delegated authority

### 4.3 Architecture & Engineering
- Define and maintain cloud security architecture standards (paved-road)
- Approve high-risk architectural exceptions
- Sign off security gates in CI/CD

### 4.4 Operations
- Oversee CSPM, IAM, KMS, SIEM operations for cloud
- Operate the Shared Responsibility Matrix
- Lead cloud-incident response (jointly with SOC)

### 4.5 Compliance & Assurance
- Coordinate ISO/IEC 27017/27018 certification activity
- Manage CSP attestation reviews (SOC 2, ISO, FedRAMP)
- Liaise with internal/external auditors

### 4.6 Sub-Processor Management
- Approve sub-processor additions / changes
- Operate the supplier-cloud-assurance process

### 4.7 People & Culture
- Maintain the cloud security skill matrix and training plan
- Run the Cloud Awareness Programme

## 5. Key Performance Indicators (excerpt)

| KPI | Target |
|-----|--------|
| Cloud SoA controls implemented | 100% by end Y1 |
| High/Critical cloud risks treated within SLA | ≥ 95% |
| Sub-processor reviews up-to-date | 100% |
| CSPM critical findings open > 14 days | 0 |
| Customer security questionnaires turnaround | ≤ 5 business days |

## 6. Required Competencies

- Cloud architecture experience (minimum 5 years across ≥ 2 CSPs)
- Recognised certification (CCSP, CISSP, CCSK, AWS/Azure security)
- Familiarity with ISO/IEC 27001/27017/27018 and at least one regional privacy law
- Incident-response leadership experience

## 7. Resources

- Direct team: cloud-security engineers, GRC analysts (initial sizing 5 FTE)
- Tooling: CSPM, SIEM, PAM, KMS, vulnerability scanner
- Budget: per annual security plan

## 8. Conflicts of Interest

The CSO is independent of:
- Engineering delivery KPIs (no shared OKRs that compromise security)
- Sales targets

## 9. Review

This Terms of Reference is reviewed annually and on any organisational change.
